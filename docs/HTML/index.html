HTML Review Notes

HTML Basics

  • Role of HTML: HTML is the foundation of web structure, defining the elements of a webpage.
  • HTML Elements: Represent content using opening and closing tags (e.g., <h1></h1>).
  • HTML Structure: Consists of a <head> and <body>.
  • Common Elements: Headings (<h1> to <h6>), paragraphs (<p>), containers (<div>).
  • Void Elements: Elements without a closing tag (e.g., <img>).
  • Attributes: Add metadata and behavior to elements.

Identifiers and Grouping

  • IDs: Unique identifiers.
  • Classes: Grouping for styling and behavior.

Special Characters and Linking

  • HTML Entities: e.g., &amp;, &lt;.
  • Link element: For external stylesheets.
  • Script element: For external JavaScript.

Boilerplate and Encoding

  • HTML Boilerplate: Includes <!DOCTYPE>, <html>, <head>, and <body>.
  • UTF-8 Encoding: Ensures universal character display.

SEO and Social Sharing

  • Meta tags: Affect SEO.
  • Open Graph tags: Improve social media previews.

Media Elements and Optimization

  • Replaced Elements: Like <img> and <iframe>.
  • Media Optimization: Improve performance and compatibility.
  • Image Formats and Licenses: Know types and usage rights.
  • SVGs: Scalable vector graphics.

Multimedia Integration

  • Audio & Video: Use <audio> and <video>.
  • <iframe>: Embed external content (e.g., YouTube).

Paths and Link Behavior

  • Target attribute: Controls link behavior (e.g., _blank).
  • Absolute vs Relative Paths
  • Path Syntax: /, ./, ../.
  • Link States: Hover, active, etc.

Semantic HTML

Structure and Elements

  • <header>: Introductory content
  • <nav>: Navigation links
  • <article>: Self-contained content
  • <aside>: Sidebar or related content
  • <section>: Related content grouping
  • <footer>: Footer section

Other Semantic Tags

  • <em>, <strong>, <i>, <b>
  • <dl>, <dt>, <dd>
  • <blockquote>, <q>
  • <abbr>, <address>, <time>
  • <sup>, <sub>, <code>
  • <u>, <ruby>, <s>

HTML Forms and Attributes

Form Structure:

<form method="POST" action="submit-url">
  <input type="text" name="name" placeholder="Your Name" required />
</form>

Common Attributes: type, name, placeholder, required, disabled, readonly

Labels:

<label for="email">Email:</label>
<input type="email" id="email" name="email" />

Radio and Checkbox Example:

<fieldset>
  <legend>Choose Options</legend>
  <label><input type="checkbox" name="option" value="1" /> One</label>
</fieldset>

Buttons:

<button type="submit">Submit</button>

HTML Tables

<table>
  <caption>Grades</caption>
  <thead><tr><th>Name</th></tr></thead>
  <tbody><tr><td>Alex</td></tr></tbody>
  <tfoot><tr><td>Average</td></tr></tfoot>
</table>

HTML Tools

  • HTML Validator: Checks for valid syntax.
  • DOM Inspector: Examine live HTML in browser.
  • DevTools: Browser debugging tools.

Accessibility Overview

WCAG Principles (POUR)

  • Perceivable
  • Operable
  • Understandable
  • Robust

Assistive Technologies

  • Screen readers, Braille keyboards, Voice recognition, etc.

Best Practices

  • Use correct heading levels
  • Use <label> with form inputs
  • Provide alt text for images
  • Use descriptive link text
  • Include captions/transcripts for multimedia

WAI-ARIA and Roles

  • ARIA Roles: role="alert", role="menu", etc.
  • aria-label / aria-labelledby: Provide accessible names.
  • aria-hidden: Hide decorative elements.
  • aria-expanded: Toggle visibility state.
  • aria-live: Announce live updates.
  • aria-controls / aria-describedby: Element associations.
